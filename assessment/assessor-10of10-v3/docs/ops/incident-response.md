# Incident Response

## Objectives
- Restore service quickly and safely
- Preserve evidence
- Notify stakeholders appropriately

## Severity
- SEV1: complete outage, data loss risk, security incident
- SEV2: partial outage, degraded performance
- SEV3: minor issue, workaround exists

## Process
1. Detect and declare severity
2. Assign incident commander
3. Contain and mitigate
4. Communicate updates on schedule
5. Post-incident review (PIR) with actions tracked

## Evidence
- Incident log, timeline, root cause, corrective actions.
